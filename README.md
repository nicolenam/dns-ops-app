# DNS Ops App

A small Streamlit app for generating DNS records, customer-facing email copy, and rollout checklists for common DNS operations.

## What it does

The app supports these workflows:

- `SendGrid + certificate prep`
- `SendGrid only`
- `Certificate only`
- `Subdomain go-live`
- `Main domain go-live`
- `WordPress update`

It can generate:

- DNS record output
- customer email draft text
- internal checklist text

## Project files

- `app.py` - Streamlit UI and workflow logic
- `integrations.py` - SendGrid/AWS integration and DNS helper logic
- `AWS_SENDGRID_INTEGRATION.md` - implementation outline for the provider integrations

## Deploy on Streamlit Community Cloud

This app can be deployed directly from GitHub on Streamlit Community Cloud.

Files needed for deployment:

- `app.py` - app entrypoint
- `requirements.txt` - Python dependencies for the hosted environment

Deployment steps:

1. Push the repository to GitHub.
2. Go to `https://share.streamlit.io/`.
3. Click `Create app`.
4. Select:
   - Repository: `nicolenam/dns-ops-app`
   - Branch: `main`
   - Main file path: `app.py`
5. In `Advanced settings`, choose a Python version if needed and add secrets.
6. Click `Deploy`.

Demo mode secrets:

```toml
DNS_OPS_USE_MOCK_MODE="true"
```

Live mode secrets:

```toml
DNS_OPS_USE_MOCK_MODE="false"
SENDGRID_API_KEY="your-sendgrid-api-key"
SENDGRID_REGION="global"
SENDGRID_SUBDOMAIN=""
DMARC_POLICY="reject"
DMARC_RUA="example@rep.dmarcanalyzer.com"
DMARC_RUF="example@for.dmarcanalyzer.com"
```

Notes:

- This repository currently supports real SendGrid integration in live mode.
- AWS ACM live integration is not implemented yet, so certificate generation will still fail in live mode.

## Mock mode vs live mode

The app supports two modes controlled by `DNS_OPS_USE_MOCK_MODE`.

### Mock mode

Use this for demos or placeholder output:

```bash
cd /Users/eunjungnam/dns-ops-app
DNS_OPS_USE_MOCK_MODE=true ./.venv/bin/streamlit run app.py
```

### Live mode

Use this when you want the app to call real provider integrations where available:

```bash
cd /Users/eunjungnam/dns-ops-app
DNS_OPS_USE_MOCK_MODE=false ./.venv/bin/streamlit run app.py
```

## Current integration status

### SendGrid

`SendGrid only` and the SendGrid portion of `SendGrid + certificate prep` can use the real SendGrid Domain Authentication API in live mode.

Required environment variable:

```bash
SENDGRID_API_KEY=your-sendgrid-api-key
```

Optional environment variables:

```bash
SENDGRID_REGION=global
SENDGRID_SUBDOMAIN=
DMARC_POLICY=reject
DMARC_RUA=example@rep.dmarcanalyzer.com
DMARC_RUF=example@for.dmarcanalyzer.com
```

Notes:

- The current SendGrid logic is designed to match a simple manual flow: use defaults, do not choose advanced settings, and manually verify after DNS records are added.
- If `SENDGRID_SUBDOMAIN` is left unset, the app does not force a custom subdomain.
- DMARC is currently generated by the app as a default TXT value.

### AWS ACM

AWS certificate integration is not implemented yet in live mode.

That means:

- `Certificate only` will still fail in live mode
- `SendGrid + certificate prep` will only be partially live until AWS ACM integration is added

## What the UI shows

The generated records panel is formatted as a simple manual-install table with:

- `Type`
- `Name`
- `Value`

For SendGrid, this includes:

- mail `CNAME`
- `s1._domainkey` `CNAME`
- `s2._domainkey` `CNAME`
- `_dmarc` `TXT`
